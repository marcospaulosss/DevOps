ReadAll
GET/api/shelves{?search,sort,page,per_page}

O parametro ‘search’ segue o formato: “(field[operator]:‘value’)”. Exemplos:

  • title[eq]:‘first shelf’: O campo title dever possuir o valor exatamente igual à first shelf

  • title[ne]:‘first’: O o valor do campo title deve ser diferente de first

  • title[contains]:‘first’: O campo title deve conter o valor first

  • title[icontains]:‘first’: O campo title deve conter o valor first ignorando case sensitive

  • created_at[gt]:‘2019-01-01’: O valor do campo created_at deve ser maior que 2019-01-01

  • created_at[gte]:‘2019-01-01’: O valor do campo created_at deve ser maior ou igual que 2019-01-01

  • created_at[lt]:‘2019-01-01’: O valor do campo created_at deve ser menor que 2019-01-01

  • created_at[lte]:‘2019-01-01’: O valor do campo created_at deve ser menor ou igual que 2019-01-01

Example URI

GET severino.estrategia.dev/api/shelves?search=(created_at[gte]:'2019-01-01')&sort=+created_at&page=1&per_page=5
URI Parameters
HideShow
sort
string (optional) Example: +created_at

Ordena o resultado pelo campo determinado. A ordem varia entre ASC (adicionando o prefixo +) ou DESC (adicionando prefixo -).

Choices: +id -id +created_at -created_at +title -title

page
string (optional) Example: 1

Número da página. Se for 0 a paginação vai ser ignorada.

per_page
number (optional) Example: 5

Total de itens por página. Se for 0 a paginação vai ser ignorada.

search
string (required) Example: (created_at[gte]:'2019-01-01')

Os parâmetros de busca devem estar entre parenteses. O separador , funciona como o operador OR e + como AND.

ReadAll
GET/api/albums{?search,sort,page,per_page}

O parametro ‘search’ segue o formato: “(field[operator]:‘value’)”. Exemplos:

  • title[eq]:‘first album’: O campo title dever possuir o valor exatamente igual à first album

  • title[ne]:‘first’: O o valor do campo title deve ser diferente de first

  • title[contains]:‘first’: O campo title deve conter o valor first

  • title[icontains]:‘first’: O campo title deve conter o valor first ignorando case sensitive

  • created_at[gt]:‘2019-01-01’: O valor do campo created_at deve ser maior que 2019-01-01

  • created_at[gte]:‘2019-01-01’: O valor do campo created_at deve ser maior ou igual que 2019-01-01

  • created_at[lt]:‘2019-01-01’: O valor do campo created_at deve ser menor que 2019-01-01

  • created_at[lte]:‘2019-01-01’: O valor do campo created_at deve ser menor ou igual que 2019-01-01

O parametro ‘search’ segue o formato: “(field[operator]:‘value’)”. Exemplos:

  • created_at[gt]:‘2019-01-01’: O valor do campo created_at deve ser maior que 2019-01-01

  • created_at[gte]:‘2019-01-01’: O valor do campo created_at deve ser maior ou igual que 2019-01-01

  • created_at[lt]:‘2019-01-01’: O valor do campo created_at deve ser menor que 2019-01-01

  • created_at[lte]:‘2019-01-01’: O valor do campo created_at deve ser menor ou igual que 2019-01-01
